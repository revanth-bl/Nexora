# 🤝 Contributing to Nexora

First of all, thank you for considering contributing to **Nexora**! ❤️

Nexora is an open-source engineering knowledge platform built by the community, for the community. Every improvement—whether it's fixing a typo, improving an explanation, adding a new guide, or creating a complete learning path—helps thousands of learners around the world.

# 🚀 Ways You Can Contribute

You don't have to write hundreds of pages to help.

You can contribute by:

- 📝 Fixing spelling or grammar
- 📖 Improving explanations
- 💻 Adding code examples
- 🛠 Fixing broken links
- 📷 Adding diagrams or illustrations
- 🎯 Improving interview questions
- 🧪 Creating labs or projects
- 🗺 Expanding learning roadmaps
- 🌐 Translating documentation
- 🐛 Reporting bugs
- 💡 Suggesting new topics

Every contribution matters.

# 📂 Repository Structure

Every technology follows the same structure.

```text
technology/
│
├── README.md
├── commands/
├── concepts/
├── architecture/
├── best-practices/
├── design-patterns/
├── troubleshooting/
├── cheat-sheets/
├── projects/
├── labs/
├── learning-paths/
├── interview-questions/
├── faqs/
└── official-resources/
```

Please follow this structure when adding new content.

---

# ✍️ Writing Standards

Every article should answer these questions:

1. What is it?
2. Why does it exist?
3. When should I use it?
4. When should I avoid it?
5. How does it work?
6. Real-world examples
7. Common mistakes
8. Best practices
9. Related topics
10. Official documentation

Explain concepts clearly and avoid unnecessary jargon whenever possible.

---

# 📝 Markdown Style Guide

- Use meaningful headings.
- Keep paragraphs short.
- Use bullet lists when appropriate.
- Include code blocks for commands.
- Include explanations before code.
- Use tables only when they improve readability.
- Prefer relative links for internal documentation.

Example:

````text
## Overview

Explain the concept.

## Syntax

```bash
docker run nginx
```

## Explanation

Describe what the command does.
